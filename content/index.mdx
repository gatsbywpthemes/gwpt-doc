# 🚀 All that you need to get it running.

## Setting up your WordPress Website.

You will need a running WordPress installation. Your content comes from your WordPress website.
Gatsby fetches the data from your WordPress website.

### Automated way

You will use one of our WordPress starters. They come with all the required and recommended plugins.
Each starter contains all the required and recommended plugins, as well as the parent and the child theme
(all of our Gatsby themes work with the same WordPress parent theme and a specific WordPress child theme).

You will also install some content (post and pages) to start even more quickly.  
To have a quick all-in-one installation experience, you will use a free, third party WordPress plugin <a href="https://wordpress.org/plugins/all-in-one-wp-migration/" target="_blank" rel="noopener noreferrer">All In One WP Migration</a> third-party plugin.

- Install WordPress on your hosting
- Install and activate _All-in-One WP Migration_ plugin that is available in the official WordPress repo.
- Click **Import** in the _All-in-One WP Migration_ menu
- The starter size may exceed the _Maximum upload file size_ available (this may vary depending on your hosting).
  _All-in-One WP Migration_ gives you a few solutions here. An easy way to upgrade the import size to `512MB` is by <a href="https://import.wp-migration.com/all-in-one-wp-migration-file-extension.zip">downloading and installing this add-on plugin.</a>
- Chose **Import from File** and the `.wpress` package file corresponding to the Gatsby example that you are planning to use.

  <img src="./import.png" />

- Our starter is now installed!
- Login to your website. You will have to use the temporary credentials that have just been set up:
  - username: `admin`
  - password: `gatsbyWpThemes`
- Go to **Settings > Permalinks** and save your permalinks structure. All you have to do here is to click the **Save Changes** button.
- Create a new user with secure credentials, login with your new credentials and delete the temporary admin account.

  That's it!

### Manual way

- Install the required themes: the gatsby-wp one, and the child theme corresponding to the gatsby theme you want to use.(they are all in the package that you have received in your welcome mail)
- Install the required plugins

  - <a href="https://github.com/gatsbyjs/wp-gatsby" target="_blank">
      WPGatsby
    </a>
  - <a
      href="https://github.com/wp-graphql/wp-graphql/tree/release/v0.9.1"
      target="_blank"
    >
      WPGraphQL
    </a>

* <strong>Permalinks structure</strong>

  In order that the plugin works correctly, your WordPress installation must
  have pretty permalinks enabled (commonly provided by the majority of hosting
  services). Also, The permalink structure **must not** be the default (plain,
  `http://example.com/?p=123.`) WordPress permalink structure. This can be done
  via your WordPress admin area in Settings > Permalinks.

## Setting up your Gatsby Site.

### Set Up Your Development Environment

go to <a href="https://www.gatsbyjs.org/tutorial/part-zero/" target="_blank">Gastby documentation</a> and prepare your computer

### Installing your Gatsby starter

Choose the starter you want to use among the ones you have received inside your welcome mail

- If you are going to build something totally custom, we recomend using the starter data: it looks exactly like the starter, but only the gatsby-theme-blog-data is installed as theme, all the code from the gatsby-theme-wp-blog-starter is in the site, so you can modify everything easely without shadowing, while you have all your components very well organized

1. Unzip your starter.
2. Open the `.nprmc` file and replace `YOU-AUTH-TOKEN` string by your token that you can find in your welcome mail
3. Run `yarn` command from your starter directory, in order to install all the npm modules
4. Configure your Gatsby site.
   Most importantly, provide the url of your WordPress source website.
5. Run `gatsby develop` to start the developement server

### Configure your Gatsby site

```bash
├── ✏️ .env
├── README.md
├── ✏️ config.js
├── ✏️ configColors.js
├── ✏️ configSizes.js
├── ✏️ configTypography.js
├── gatsby-config.js
├── package.json
├── src
│   ├── 🗂 @chakra-ui
│   ├── 🗂 @gatsbywpthemes
│   └── 🗂 styles
└── 🗂 static
    ├── 🔃 favicon.svg
    ├── 🔃 dark-mode-logo.svg
    └── 🔃 logo.svg
```

In order to setup your Gatsby website, you might need to edit the configuration files ( ✏️ ) located in the root of your project.

- ✏️ `.env` - basic configuration
- ✏️ `config.js` - basic configuration
- ✏️ `configColors.js` - if you want to change the theme colors
- ✏️ `configSizes.js` - if you want to change some layout values (responsive breakpoints, container width, etc.)
- ✏️ `configTypography.js` - if you want to modify the typography settings (font family, heading sizes etc.)

#### Setting up the `.env` file

The easiest way to setup your `.env` file is to rename the existing `.env-sample` to `.env`.

```
# rename this file to .env and supply the values listed below
# also make sure they are available to the build tool (e.g. Netlify)
# notice: variables prexifed with GATSBY_ will be made available to client-side code
# GATSBY_WP_URL=the_url_of_your_wordpress_website
# GATSBY_MAILCHIMP_ENDPOINT=https://gmail.us20.list-manage.com/subscribe/post?u={your_value}&amp;id={your_id}
```

**Your WordPress website**

Uncomment the `GATSBY_WP_URL` line and replace the `the_url_of_your_wordpress_website` by the url of your WP website.

example:

```
GATSBY_WP_URL=https://example.com
```

**Mailchimp endpoint** <a name="mailchimp"></a>

You can subscribe new email addresses to a Mailchimp email list with our Newsletter widget.  
In order to activate Mailchimp, assign your Mailchimp list endpoint URL to the environmental variable `GATSBY_MAILCHIMP_ENDPOINT`

Our themes use [`gatsby-mailchimp-plugin`](https://www.gatsbyjs.org/packages/gatsby-plugin-mailchimp) for the MailChimp subscriptions.
Check [the plugin documentation](https://www.gatsbyjs.org/packages/gatsby-plugin-mailchimp/#mailchimp-endpoint) for more information on how to locate your Mailchimp endpoint.

example:

```
GATSBY_MAILCHIMP_ENDPOINT=https://gmail.us20.list-manage.com/subscribe/post?u=xxxyyyzzz&amp;id=xxyyzz
```

#### Further customization with `config.js`

The default options are listed below.
We discuss each one in details in the Options section.

```javascript
const config = {
  pathPrefix: "",
  paginationPrefix: "page",
  addWordPressComments: true,
  menuName: "main",
  siteUrl: "https://example.com",
  author: "Name Placeholder",
  social: [
    {
      name: "twitter",
      url: "https://twitter.com/gatsbyjs",
    },
  ],
  fonts: ["Abril Fatface", "Fira Sans"],
  gaTrackingId: 0,
  googleTagManagerId: 0,
  addSiteMap: false,
  siteMapOptions: {},
  skipTitle: [],
}
```

### Options

**wordPressUrl** (required)
Provide a url to your WordPress source website

---

**pathPrefix** (optional)
`(default: "")`

Typically, your Gatsby website will be hosted at the root of its domain. In that case, the pathPrefix is an empty string and you can skip this setting.
You will need to set the pathPrefix though, if your gatsby website is hosted at something other than the root (/), for example `https://example.com/demo`

Adding the path prefix requires two steps:

1. setting the pathPrefix (make sure to preceed it with a slash)
   example:

```javascript
pathPrefix: "/demo"
```

2. adding `--prefix-paths` flag when building your website:

```bash
gatsby build --prefix-paths
```

---

**paginationPrefix** (optional)
`(default: 'page')`

Prefix for paginated content.

What is the url structure of any paginated content on your WordPress website? By default, WordPress uses `page` as the prefix, that means it preceeds page numbers by the `page` keyword (`page/2`, `page/3`, ...).
Your Gatsby permalinks structure should match your WordPress structure. Skip this setting, unless your changed the pagination url format on your WordPress site.

---

**addWordPressComments** (optional)
`(type: Boolean, default: true)`

Whether WordPress comments should be activated.

Our themes come with the support for native WordPress comments. This feauture is activated by default.  
The comments are displayed for posts that have comments status set to "Allow Comments".
Commenting on your Gatsby site works similarily to commenting on your WordPress site.
The main difference is that **we only support two levels of comments nesting.**  
Under the hood, comments are fetched from WordPress and updated on WordPress (and refetched if necessary) with Apollo Client.
Consequently, some of your WordPress Discussion settings applies to your Gatsby comments, in particular: email notifications, moderation and blocking rules.

You can deactivate WordPress comments this by changing `addWordPressComments` to `false`.

example:

```javascript
addWordPressComments: false,
```

---

**menuName** (optional)
`(type: String, default: "main")`

The WordPress name of the navigation menu that will be used.

You can choose any of the menus that you had created on your WordPress site by passing its name to the **menuName** option.

![Menu name set to Main Menu on WordPress](https://wptemplates.pehaa.com/docs/gatsby-themes/assets/menuname.png)

example:

```javascript
menuName: "Main Menu",
```

---

**siteUrl** (obligatory for a production website)  
`(type: Url, default: "https://example.com")`

You can use the placeholder value `https://example.com` in the developement phase but once your site is built for production, you have to provide the url of your destination site.

This is an essential setting for social share buttons (in Posts), SEO open graph data,
and twitter card image. It's also necessary if you opt for an xml sitemap.

---

**gaUniversalTrackingId** (optional, deprecated)  
`(type: String)`

You can add Google Analytics to your Gatsby site by
providing your Google Analytics tracking ID.
Also, please note that the Google Analytics functionality is disabled in the developement mode.

example:

```javascript
gaUniversalTrackingId: "UA-111111111-1",
```

**gaUniversalOptions** (optional, deprecated)  
`(type: Object, default: {})`

Our themes use the official Gatsby plugin [`gatsby-plugin-google-analytics`](https://www.npmjs.com/package/gatsby-plugin-google-analytics) and
you have access to all [its options.](https://www.npmjs.com/package/gatsby-plugin-google-analytics#how-to-use)

```javascript
gaUniversalTrackingId: "UA-111111111-1",
gaUniversalOptions: {
  head: true,
},
```

### `static` folder

If you want to use a custom logo and/or a favicon, you need to put the files into the `static` folder in the root of your example. The `static` folder is already there.

**logo** (optional)  
`(type: String)`

The name of your logo file, this file must be located in the `static` folder. If no logo is provided, the site title is displayed instead.

example:

```bash
└── 🗂 static
    ├── 🔃 favicon.svg
    ├── 🔃 dark-mode-logo.svg
    └── 🔃 logo.svg
```

```javascript
logo: "logo.svg",
```
